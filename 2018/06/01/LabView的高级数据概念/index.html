LabView的高级数据概念

发表于 2018-06-01 | 分类于 LabView |

多态VI

多态VI就是通过自动适应所连线的数据类型，能够处理不同数据类型的输入。可以在单一输入或输出端子上接受不同的数据类型。

在一个多态子VI 上右键，选择 visible item >> polymorphic VI selector可以显示多态VI选择器。多态VI选择器是一个下拉列表，显示属于多态VI的所有成员VI。

除了多态选择器，也可以在子VI上右键选择Select Type>>子菜单下的各个选项。

高级文件I/O：文本文件、二进制文件和配置文件

LabView中文件的读写函数是多态的，可以通过文件路径输入或者文件号输入使用这些函数。

使用Open/Create/Replace File函数可以输出文件号，当读写文件完成后，将文件号传递给Close File函数，告诉LabView不需要访问该软件。

高级文件函数

在Programming >> File I/O >> Advanced File Functions中有很多文件I/O函数。

文件位置：

文件标记是一个数字，表示当前工作文件的字节偏移量。

当读写文件时，LabView自动调整文件标记，也可以使用Set File Position函数指定文件标记的值。使用Get File Position函数可以获取当前文件标记值。

文件结尾：

文件结尾是文件最后一个字节的偏移量，也就是文件的大小，使用Set File Size和Get File Size函数可以设置和得到文件结尾。

当设置文件结尾的时候，如果设置值大于当前文件结尾的值，那么会自动补0；如果小于当前文件结尾的值，那么会把多余的内容删除掉。

文件和文件夹的移动、复制、删除都有相应的函数可以使用。

配置文件

配置文件VI（Programming >> File I/O >> Configuration File VIs）允许用户访问一种特殊的文件格式，就是键值对。配置文件的扩展名通常为“.ini”。

使用Open Config Data.vi可以打开配置文件的文件号。使用Close Config Data.vi可以关闭配置文件的文件号。

从配置文件中读写键值对，可以使用函数Write Key.vi 和Read Key.vi。

调用其他语言的代码

LabView提供了3种方式来调用编译过的外部代码：

  • 通过System Exec.vi使用命令行方式；

是最简单的选项，可以根据需要打开一段独立可执行的代码或程序。Connectivity >> Libraries&Executables.

  • 使用Call Library Function Node，调用动态链接库或者DLL；

如果有动态链接库，windows上的动态链接库（.dll）、Linux上的共享库或共享对象（.so）、Mac OS X上的框架（.framework）,可以在Labview中使用Call Library Function Node调用。

  • 使用Code Interface Node与编译过的代码交互。

如果想要编写自己的C代码，并将其嵌入Labview中，应该使用Code Interface Node。

DLL是存放在.dll文件中编译过的小型库文件，该文件封装了一些功能代码和一组供其他应用程序调用的接口函数。

通常需要DLL提供完善的文档以方便使用，文档中对包含的函数和函数与应用程序间如何传递数据进行介绍。
