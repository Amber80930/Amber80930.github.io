Vue常用命令

发表于 2018-11-27 | 分类于 前端框架 |

1. v-if

条件渲染指令，根据其表达式的Boolean值判断是否渲染，只渲染其身后表达式为true的元素。

2. v-show

条件渲染指令，元素v-show值为false时，元素会隐藏，但是只是相当于修改元素css属性，使display：none，元素还是会被编译。

而v-if才能称为实际意义上的条件渲染，若表达式初始值为false，则元素一开始不会被渲染，只有当条件表达式变为true之后才开始编译。

v-if更适合条件不经常改变的场景，因为其切换开销较大，v-show适用于频繁切换条件。

3. v-else

v-else必须跟在v-if/v-show指令之后，否则不起作用。

如果v-if/v-show指令表达式为true，则else元素不显示；如果v-if/v-show指令表达式为false，则else元素显示在页面上。

4. v-for

v-for类似于JS中的遍历，用法是：v-for = “item in items”。其中items是数组，item是数组中的元素。

为了给Vue一个提示，以便它能跟踪每个节点的身份，从而重用和重新排序现有元素，你需要为每项提供一个唯一key属性。即：v-for=“（item,key） in items”。

5. v-bind

绑定属性，比如绑定某个class元素或元素的style样式。使用方法是v-bind：class =“”。

可以缩写为“：”。

v-bind 的三种修饰符：

l .sync : 用于组件props属性，将父组件和子组件的值进行双向绑定，两个组件共享这一数据，无论谁修改了数据，组件获取的数据都是一致的。

l .once : 用于组件props属性，但是进行的是单次绑定，即将绑定数据传递给子组件之后，子组件单独维护这个数据，和父组件数据再无关系。

l .camel : 将绑定特性的名称变回驼峰命名（第一个单词首字母小写，第二个单词以后首字母大写）

6. v-on

绑定事件，用于绑定指定元素的监听事件，比如点击事件，v-on：click = “事件方法名称”。

可以缩写为“@”。

v-on的修饰符包括：

l .stop ： 阻止事件冒泡（分别给父元素和子元素绑定了事件，点击子元素的同时会 触发父元素的绑定事件）

l .prevent ：阻止默认事件

l .capture ：添加事件监听器

l .slef ：当事件是从监听元素本身触发时才触发回调

除了事件修饰符之外，v-on还提供了按键修饰符，如：<input v-on : keyup.enter = “submit” />.

7. v-model

将组件与变量进行双向绑定，当组件数据修改时，变量会随之改变。常用于input、select、textarea标签中。

后面可加修饰符，如.lazy，只有标签中的值发生改变时 才同步到变量。.number会自动将用户输入转为Number类型。

8. v-text

向标签内注入数据，并且覆盖标签内的原来内容。

如果直接使用<span> \</span>，页面会出现闪现信息的情况，

但是用<span v-text =”msg” > </span> 就不会出现这种情况。

9. v-html

向标签内注入HTML内容作为其子元素。

10. v-once

被修饰的标签中的数据不允许修改。

11. v-el

获取DOM对象。为DOM元素注册了一个索引，使我们可以利用其所属实例的 ”$els” 属性调用。

如：

或者在vm内部通过this调用。

12. v-ref

获取组件对象。实例可以通过“$refs”访问子组件。

利用vm.$refs.title 可以得到组件的实例。

13. v-pre

跳过编译当前元素和子元素，只显示原始的标签，用来减少编译时间。

如：<div v-pre> </div>，这样不会编译，直接在网页原样显示。

14. v-cloak

相当于在元素上添加了一个 [v-cloak] 属性，直到关联实例结束编译。官方推荐和css规则 [v-cloak]{ display ：none} 一起使用，可以隐藏未编译的标签直到实例准备完毕。

15. 自定义指令

15.1 指令的注册

全局注册指令：Vue.directive ( id, definition )

局部自定义指令：

该指令就只能在当前组件内通过v-local-directive的方式调用，无法被其他组件调用。

15.2 指令的定义对象（钩子函数）

在注册指令时，可以传入定义对象。定义对象主要包括三个钩子函数：bind、update、unbind。

  • bind：在指令第一次绑定到元素上时调用，只被调用一次。
  • update：在bind之后以初始值为参数进行第一次调用，之后每次绑定值发生改变时调用。有两个参数分别是：newValue、oldValue。（Vue.js 2.0 修改： bind函数执行后不直接调用update函数； 只要组件发生重绘，无论指令接受值是否改变都会调用update函数）
  • unbind：指令从元素上解绑时调用，只调用一次。
  • componentUpdate （Vue.js 2.0新增）： 当整个组件都完成了update状态后（所有DOM都更新后）调用该钩子函数。

另外，如果只需要使用update函数时，可以直接传入一个函数代替定义对象。

15.3 指令实例属性

在指令的钩子函数内，可以通过this来调用指令实例属性。指令的实例属性包括：

  • el：指令绑定的元素
  • vm：该指令上下文的ViewModel，可以为new Vue() 的实例，也可以是组件实例
  • expression: 指令的表达式，不包括参数和过滤器
  • arg：指令的参数
  • name：指令的名字，不包括v-前缀
  • modifiers：一个对象，包含指令的修饰符
  • descriptor：一个对象，包含指令的解析结果

Vue.js 2.0 中取消了指令实例这一概念，即在钩子函数中不能用this调用指令的相关属性了，指令的相关属性均通过参数的形式传递给钩子函数。

15.4 元素指令

普通指令需要绑定在某个具体的DOM元素上，但是元素指令可以单独存在。例如：

16. 指令的高级选项

16.1 params

定义对象中可以接受一个params数组。

除了直接传入数值外，params还支持动态绑定数据，可以设置一个watcher监听，当数据发生变化时，就会回调这个函数。

16.2 deep

用来监听对象内部发生的变化。

16.3 twoWay

如果需要向Vue实例写回数据，及需要在定义对象中使用twoWay：true，这样可以在指令中使用this.set ( value)来写回数据。

16.4 acceptStatement

设置为true时，允许自定义指令接受内联语句，同时update函数的值是一个函数，在调用该函数时，它将在所属实例作用域内运行。

“i++”就是内联语句，fn是一个函数。如果在update函数中，运行fn( )，则会执行内联语句i++，此时vm.i = 1。但是vm.i并不会触发update函数。

16.5 terminal

作用是阻止Vue.js遍历这个元素及其内部元素，并由该指令本身去编译绑定元素及其内部元素。内置的指令v-if、v-for都是terminal指令。

16.6 priority

指定指令的优先级，普通指令默认是1000，terminal指令默认是2000。同一元素上优先级高的指令会先执行，相同优先级的指令按照顺序执行。
