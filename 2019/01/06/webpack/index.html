webpack

发表于 2019-01-06 | 分类于 前端框架 |

在webpack的概念里，一张图片、css、甚至一个字体都称为模块，彼此存在依赖关系，webpack就是来处理模块间的依赖关系，并把它们进行打包。

Webpack 的主要使用场景是单页面复应用（SPA），SPA通常是由一个html文件和一堆按需加载的js组成。

导入、导出模块

export、import用于导入和导出模块，一个模块就是一个js文件，它的作用域是独立的，里面定义的变量无法在外部访问。

模块导出后，在需要使用模块的文件用import再导入，就可以在这个文件内使用这些模块了。

使用export default 来输出默认模块，用户可以在导入时自定义名称。

webpack配置

基础配置

webpack配置中最重要也是必选的两项是入口（entry）和出口（output）。入口的作用是告诉webpack从哪里开始寻找依赖，并且编译，出口则是用来配置编译后文件的存储位置和文件名。

Entry 中的main是我们配置的单入口，webpack会从main.js文件开始工作。

Output中path选项用来存放打包后的文件输出目录，publicPath指定资源文件引用的目录，filename用于指定输出文件的名称。

完善配置

对于不同的模块，需要不同的加载器（Loaders）来处理，加载器是webpack的重要部分。通过安装不同的加载器可以对各种后缀名的文件进行处理。

比如现在写CSS样式，需要用到style-loader和css-loader，首先用npm安装：

npm install css-loader –save-dev

npm install style-loader –save-dev

安装完成后，在webpack的配置文件中配置Loader，增加对.css文件的处理。

在module的rules属性中可以指定一系列的loaders，每个loader都必须包含test和use 两个选项。这段配置的意思是，当webpack编译过程中遇到require（）或import语句导入一个后缀名为.css文件时，先将它通过css-loader转换，再通过style-loader转换，然后继续打包。

插件

项目如果很大的话，样式会很多，都放在js中太占体积，还不能缓存，这时可以使用一个extract-text-webpack-plugin插件，把散落的css提取出来，生成一个main.css文件，最终在index.html中用<link>加载。

单文件组件和vue-loader

.vue 单文件组件就是一个后缀名为.vue的文件，在webpack中使用vue-loader就可以对.vue文件进行处理。

一个.vue文件包含3部分，<template>、<script>、<style>。

Style中加scope属性，表示当前的CSS只在这个组件有效，如果不加，那么div的样式会应用到整个项目。

Vue-loader在编译vue文件时，会对<template>、<script>、<style>分别处理，所以在vue-loader选项中多了一项options来进一步对不同语言进行配置。

每个vue文件就代表一个组件，组件之间是可以相互依赖的。

在<template>内写的HTML写法完全同html文件，webpack最终会将其编译为Render函数的形式。写在style中的样式，我们已经用插件extract-text-webpack-plugin配置过了，最终会统一提取并打包在main.css中。
