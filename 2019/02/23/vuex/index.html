VUEX

在实际业务中，经常有跨组件共享数据的需求，因此VUEX就是设计来统一管理组件状态的，它定义了一系列的规范来使用和操作数据，使组件应用更加高效。

安装

npm install –save vuex

在src/ modules/下新建user.js就可以在user.js中编写与user有关的需要共享部分的信息。

其中，store相当于一个仓库，包含了应用的数据和操作过程，vuex里面的数据都是响应式的，任何组件使用同一store的数据时，只要store的数据发生变化，对应的组件也会立即更新。

state、mutations、actions

Vuex选项包含三个字段：state、mutations、actions

  • state字段：保存数据

  • mutations字段：直接修改state中的数据

  • actions字段：提交mutation，可以异步操作业务逻辑

    在组件中，来自store的数据只能读取，不能提交，不能手动改变，改变store中数据的唯一途径就是显式的提交mutations。

    在组件中，通过this.$store.commit来执行mutations。如this.$store.commit（‘SET_CODE’）。

​ 需要传参数时：this.$store.commit（‘SET_CODE’，参数），多个参数可封装成一个对象传入。

​ mutations里尽量不要异步操作数据，因为如果异步操作数据了，组件在commit后，数据不能立即改变，而且不知道什么时候会改变。

​ actions和mutations的功能类似，不同的是actions里面提交的是mutations，并且可以异步操作业务逻辑（与后台进行数据交互）。

​ actions在组件中通过$store.dispatch（‘LoginByUsername’）触发。

​ 涉及改变数据的就用mutations，存在业务逻辑的就用actions。

getter、modules

getter通常用于过滤数据，与组件中的计算属性相似。modules用来将store分割到不同的模块。
